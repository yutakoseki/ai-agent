# ブランチ命名規則

## 環境ブランチ

このプロジェクトでは、ブランチ名と環境名を完全に一致させています。

| ブランチ名 | 環境名 | 用途 |
|-----------|--------|------|
| `prod` | 本番環境 (Production) | エンドユーザー向け |
| `staging` | ステージング環境 (Staging) | QA・本番前確認 |
| `develop` | 開発環境 (Development) | 開発者の動作確認 |

## 理由

- **明確性**: ブランチ名を見れば、どの環境かすぐにわかる
- **一貫性**: コード、設定、ドキュメント全てで同じ名前を使用
- **混乱防止**: `main` が本番なのか開発なのか迷わない

## 機能開発ブランチ

| パターン | 例 | 用途 |
|---------|---|------|
| `feature/*` | `feature/123-add-login` | 新機能開発 |
| `fix/*` | `fix/456-fix-bug` | バグ修正 |
| `hotfix/*` | `hotfix/789-critical-fix` | 緊急修正（prodから分岐） |
| `refactor/*` | `refactor/cleanup-auth` | リファクタリング |
| `docs/*` | `docs/update-readme` | ドキュメント更新 |
| `test/*` | `test/add-unit-tests` | テスト追加 |

## 注意事項

- **`main` ブランチは使用しません** → `prod` を使用
- 全てのドキュメントとワークフローは `prod` を前提としています
- 既存のプロジェクトから移行する場合は、`main` → `prod` にリネームしてください

## リネーム手順（参考）

既に `main` ブランチがある場合：

```bash
# ローカルでリネーム
git branch -m main prod

# リモートに新しいブランチをプッシュ
git push origin prod

# GitHub でデフォルトブランチを prod に変更
# Settings → Branches → Default branch → prod

# 古い main ブランチを削除
git push origin --delete main
```
