# UIコンポーネントガイド

## 目的
- 共通UIコンポーネントを正として再利用し、AI生成/人手コーディングいずれでもデザインのブレを防ぐ。
- カラートークンとレスポンシブ方針を徹底し、個別スタイルの増殖を防止する。

## 共通コンポーネント（雛形）
- 配置: `apps/web/components/ui/`
- 現在の雛形
  - `Button` : バリアント `primary|ghost|secondary`、サイズ `md|sm`
  - `Card`   : タイトル/アクション/余白有無をサポート
  - `Input`  : ラベル/エラーメッセージ付き
- ギャラリー: `apps/web/app/ui-gallery/page.tsx` で一覧確認可能。新規コンポーネントを追加したらここにも反映すること。

## 利用ルール
- 色は必ずトークン経由（`primary/secondary/accent/surface/ink`）。直接HEX/RGB禁止。
- ボタン/入力/カードが必要な場合はまず既存を再利用し、足りないプロップがあれば拡張を検討する。
- バリアントを増やす場合はデザインレビューを通し、命名を最小限に保つ。
- レイアウトは Tailwind 標準ブレイクポイントでモバイルファーストに組み、余白/ガターはガイドラインの推奨値を使う（例: `px-4 md:px-6 lg:px-8`, `gap-4 md:gap-6`）。

## カラートークンの変更方法
- 変更点: `apps/web/tailwind.config.cjs` の `palette` または `:root` の `--color-*` を差し替えると、共通コンポーネント経由で全体に反映される。
- 追加トークンを増やす前に既存の `primary/secondary/accent/surface/ink` で表現できないか検討する。

## コーディングガイドとの関係
- ファイル長の目安（200行程度）と責務分離は `docs/architecture/コーディングガイドライン.md` を参照。
- コンポーネントが肥大化する前に `components/ui` へ分割し、hooks/サービス層へロジックを移す。
