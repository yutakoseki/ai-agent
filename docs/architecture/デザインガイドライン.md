# デザインガイドライン

## 目的
- ダークベースかつ鮮やかなアクセントのフラットスタイルを全体で統一する。
- `primary / secondary / accent / surface / ink` のトークン差し替えだけで配色を一括変更できるようにする（tailwindcss を正としてカラーコードの直書きを禁止）。

## カラートークン
- 定義: `apps/web/tailwind.config.cjs`（CSS 変数 `--color-*` と Tailwind トークンへマップ済み）
- トークン
  - `primary` (`--color-primary`, light/dark)
  - `secondary` (`--color-secondary`, light/dark) … ベース背景（黒系）
  - `accent` (`--color-accent`, light/dark) … 強調用
  - `surface` (`--color-surface`, raised) … パネル/カード
  - `ink` (`--color-ink`, muted/soft) … テキスト
- 使用例
  - 背景: `bg-secondary`, `bg-surface`, `bg-surface-raised`
  - テキスト: `text-ink`, `text-ink-muted`
  - 強調: `bg-primary`, `text-primary`, `border-accent`
  - シャドウ: `shadow-panel`, `shadow-brand`
  - 装飾: `bg-auth-shell`, `bg-dot-grid`
- 配色変更手順: `tailwind.config.cjs` の `palette` または `:root` の `--color-*` を変更するだけで全体に反映。

## トーン & スタイル
- ベース: 黒系 (`secondary`) のダークテーマ。
- アクセント: 鮮やかな `primary` / `accent` をボタン・リンクなど要所に限定。
- スタイル: フラット基調。シャドウは最小 (`shadow-panel`) で階層のみを示す。

## タイポグラフィ
- フォント: `font-body` / `font-heading`（`tailwind.config.cjs`）。
- テキスト色: 基本 `text-ink`、補足 `text-ink-muted`、プレースホルダや無効状態 `text-ink-soft`。

## コンポーネント実装ルール
- カラー指定は必ずトークン経由（例: `text-primary`, `bg-secondary`, `border-surface`）。HEX/RGB の直接指定は禁止。
- ステート差分はトーン内で表現（例: `hover:bg-primary-light`, `focus:ring-primary`）。
- 背景は `secondary`/`surface`、テキストは `ink` でコントラストを確保。
- アイコンやライン要素も `ink` 系または `primary` を使用し、グレーのハードコードを避ける。
- 共通 UI コンポーネント（`apps/web/components/ui/*`）を優先的に利用し、色・余白・角丸・フォーカスリングの散逸を防ぐ。新規バリアントを増やす前に既存のバリアントに寄せること。

## レスポンシブデザイン
- 方針: モバイルファースト。Tailwind のブレイクポイント（`sm 640px / md 768px / lg 1024px / xl 1280px / 2xl 1536px`）を標準とし、必要最小限の上書きに留める。
- レイアウト:
  - コンテナ: `max-w-screen-lg` 以上で中央寄せ、余白は `px-4 md:px-6 lg:px-8` を基本。
  - グリッド/カード: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` を基準に、ガターは `gap-4 md:gap-6`。
  - ナビ/ツールバー: モバイルでは縦積み→`md` 以上で水平配置。
- タイポ/余白:
  - 行間と余白はスケールで統一（例: `space-y-4`, `md:space-y-6`）。文字サイズは `text-base md:text-lg` など最小限の段階的拡張に留める。
  - ボタン/入力の高さは `h-10 md:h-11` 程度で一貫させ、タップ領域を確保。
- メディア:
  - 画像/イラストは `w-full h-auto object-contain` を基本とし、ラッパーでアスペクトを管理する。
  - アイコンは `text-ink` または `text-primary` で一貫。

## アクセシビリティと可読性
- コントラスト: `secondary` × `ink` を基準に、ボタンは `primary` 背景＋`ink` テキストで可読性を確保。
- フォーカス: `focus-visible` + `ring-primary` を必須。アウトライン除去のみは禁止。

## 今後の拡張
- ライトテーマ対応時は `:root` と `body[data-theme="light"]` で CSS 変数を分離し、Tailwind トークンは共通利用。
- 共通 UI プリセット（ボタン/カード/入力など）を `apps/web/components/ui/` にまとめてクラスの再利用性を高める。
