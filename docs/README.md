## 🔄 ドキュメントの更新

ドキュメントは常に最新の状態に保つようにしてください。

### 更新が必要な場合

- 新機能を追加した → `00-開発ロードマップ.md` を更新
- ブランチ戦略を変更した → `06-ブランチ戦略リファレンス.md` を更新
- テストコマンドを追加した → `08-テストガイド.md` を更新
- CI/CD設定を変更した → 該当するドキュメントを更新

### 更新手順

1. ドキュメントを編集
2. コミットメッセージに `docs:` プレフィックスを付ける
3. PRを作成してレビュー

```bash
git add docs/
git commit -m "docs: CI/CDセットアップガイドを更新"
git push origin feature/update-docs
```
